What is this?

This application allows you to create and spend from an Ethereum 2/3-multisig smart contract using your hardware wallet.

It was built by Unchained Capital to help holders worldwide safely store their ETH using multisig cold-storage. If you are storing ETH on a Trezor and you trust at least two other people, you could be doing it better. Now you can use this application with your trusted friends, family, and colleagues to protect your wealth.

To learn more about the design philosophy of this application and smart contract, read the blog post. If you are a developer you may want to check out the source code for the smart contract (which includes this dApp) on GitHub. Unchained Capital is also operating a bug bounty around this contract & dApp.

How does it work?

This application connects Trezor hardware wallets with the Ethereum blockchain.

When creating the multisig smart contract, 3 Trezors are required. When spending, 2/3 of these same Trezors are required. The private keys used to create and spend from a contract are only present on your Trezors. This application stores no data whatsoever about you, your Trezors, your addresses, or your transactions.

This means that you are completely in control of your own security and accounting. In particular:

  • You must yourself track the addresses of the smart contracts this application creates.
  • For each such address, you must track which Trezors you used to create the address, along with which BIP32 derivation path you used on each Trezor (it is OK to reuse the same 3 Trezors & BIP32 paths).
  • You must protect these Trezors and their corresponding wallet words. 2/3-multisig is a great security improvement over singlesig but that's no excuse to get sloppy about the security of an individual key.

Connecting to the Ethereum network

This application requires a funded account in a synced, connected Ethereum client to pay for gas.

The Ethereum transactions required to create and spend from the smart contract will be authored by this application. Broadcasting them to the Ethereum network requires a synced, connected Ethereum client. The client should have an account with some modest ETH balance to pay for gas. As of this writing, 0.01 ETH is more than sufficient.

The account used to pay for gas has absolutely no special rights to fund, spend from, or otherwise control the smart contract this application creates. Only the private keys on the 3 Trezor devices you used to create the smart contract matter. Creating and spending from a smart contract can be done by totally separate accounts, as long as this application and the same 3 Trezors are used.

Because of this design, a browser-based wallet such as Metamask provides acceptable security and is also extremely convenient.

Full Ethereum nodes such as Parity or Geth will also work, though may require some special configuration or browser extensions.

Get more details about connecting to an Ethereum client in the dApp section of the README for this GitHub repo.
